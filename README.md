# ♿ Город без барьеров

Веб-приложение на **Blazor WebAssembly (hosted, .NET 8)** для оценки доступности
объектов городской среды (вход, ширина дверей, пути, санитарная зона, информация,
парковка, персонал) с отображением на **Яндекс.Картах**.

Каждый объект оценивается по 7 критериям (0–3 балла, итог 0–21) и получает
уровень доступности:

| Баллы | Уровень | Цвет |
|-------|---------|------|
| ≥ 18  | Полностью доступно | 🟢 green |
| 9–17  | Частично доступно | 🟡 gold |
| < 9   | Недоступно | 🔴 red |

## Стек

- **Blazor WebAssembly** (клиент) + **ASP.NET Core** (сервер, хостинг WASM)
- **PostgreSQL** через **Entity Framework Core** (строка из `DATABASE_URL`,
  например Supabase/Neon). Локально без `DATABASE_URL` — **SQLite** (фоллбэк).
- **ASP.NET Core Identity** + **JWT** (3 роли: Developer / Manager / Volunteer)
- **Яндекс.Карты 2.1** (JS-interop) + клиентский геокодинг Яндекса
- CORS — разрешён любой origin (для локальной разработки WASM↔API)

## Возможности

- Карта с метками, раскрашенными по уровню доступности.
- Весь интерфейс (фильтры, кнопка добавления, статистика, форма объекта) —
  поверх самой Яндекс.Карты, а не в отдельной колонке сайта.
- Клик по карте в «режиме доступности» → форма добавления объекта
  (адрес подставляется через обратный геокодинг).
- Редактирование и удаление объектов прямо из балуна на карте.
- Поиск адреса через геокодер с перемещением карты.
- Фильтры по уровню доступности (Все / 🟢 / 🟡 / 🔴) и счётчик статистики.
- Страница **«Список»** (`/places`): таблица со всеми объектами, фильтрами,
  поиском и удалением; кнопка «Изменить» открывает форму на карте (`/?edit=<id>`).
- Сид-данные (объекты Воронежа) при первом запуске.
- Загрузка фотографий объектов (файлы хранятся в папке `uploads`, отдаются через API).
- Разграничение прав: вход по сгенерированному логину (5 симв.) и паролю
  (10 симв., буквы+цифры); Developer создаёт Manager/Developer, Manager — Volunteer.
- Окно **«Проверка»**: Manager/Developer одобряет или отклоняет поставленные метки.
- Управление пользователями по ролям (страница **«Пользователи»**).

## Запуск

```bash
dotnet build
dotnet run --project Server
```

Откройте адрес, который напечатает консоль (обычно `http://localhost:5000`).
Swagger API доступен в режиме разработки по `/swagger`.

База создаётся автоматически:

- **Локально (SQLite)** — если переменная окружения `DATABASE_URL` не задана,
  используется файл `accessibility.db` (EF Core `EnsureCreated`).
- **PostgreSQL (Supabase/Neon)** — задайте `DATABASE_URL` (см. ниже), тогда
  используется облачная БД. Строка должна начинаться с `postgresql://` (или
  `postgres://` — код сам поправит) и содержать `sslmode=require` (код добавит
  автоматически, если не указано).

При первом запуске (когда в базе ещё нет пользователей) консоль сервера печатает
**логин и пароль первого разработчика (Developer)** — сохраните их. Демо-объекты
(Воронеж) тоже подсеиваются при первом запуске.

> Локально с SQLite имена/пароли могут не совпадать с облаком — у каждой БД свой
> набор пользователей. После переключения на PostgreSQL удалите старый
> `accessibility.db`, если он мешает.

## Публикация (в публичный доступ)

Приложение — полноценный ASP.NET Core сервер (API + отдача Blazor WASM), поэтому
для публикации нужен хостинг, умеющий запускать .NET, а не только статику.

### Вариант 1. Render.com (бесплатно, через Docker)

1. Создайте аккаунт на https://render.com и подключите GitHub-репозиторий
   `pyankov5525-oss/AccessibilityMap`.
2. New → **Web Service** → выберите репозиторий.
3. Environment: **Docker** (в репозитории уже есть `Dockerfile`).
4. Остальное — по умолчанию (план Free).
5. Deploy. Через 1–2 минуты появится URL вида
   `https://accessibilitymap.onrender.com` — его и дайте руководителю.

Сервер сам берёт порт из переменной окружения `PORT` (см. `Server/Program.cs`).
Загруженные фото хранятся в файловой системе хоста; на бесплатном тарифе Render
они сбрасываются при пересборке/перезапуске (демо-данные Воронежа подсеются заново).

### База данных: PostgreSQL на Supabase (бесплатно, навсегда)

1. Зайдите на https://supabase.com → **Sign up** (через GitHub удобно).
2. Создайте организацию (например `pyankovOrg`) и проект
   (например `AccessibilityMap`). Регион — **Europe West** (близко к нам).
3. Задайте **пароль базы данных** (надёжный, сохраните его — он понадобится).
4. Когда проект будет готов: **Project Settings → Database → Connection string**
   (вкладка URI). Там будет строка вида:

   ```
   postgresql://postgres:ВАШ_ПАРОЛЬ@db.hytnewdmfhddngspxfrq.supabase.co:5432/postgres
   ```

5. Эту строку (с реальным паролем вместо `ВАШ_ПАРОЛЬ`) нужно задать как
   переменную окружения **`DATABASE_URL`** там, где запускается приложение:
   - **На Render** — в настройках Web Service → **Environment** → добавьте
     `DATABASE_URL` = полная строка. После деплоя база будет облачной, данные
     сохранятся и будут видны со всех устройств.
   - **Локально (Windows, cmd)** — перед запуском:

     ```cmd
     set DATABASE_URL=postgresql://postgres:ВАШ_ПАРОЛЬ@db.hytnewdmfhddngspxfrq.supabase.co:5432/postgres
     dotnet run --project Server
     ```

   Код сам заменит `postgres://`→`postgresql://` и добавит `sslmode=require`,
   если их нет. Первый запуск создаст таблицы и напечатает логин/пароль
   Developer в консоль.

> Пароль базы — это секрет. Не коммитьте его в репозиторий, задавайте только
> через переменную окружения.

### Вариант 2. Azure App Service (бесплатный тариф F1)

1. Создайте бесплатную подписку Azure и ресурс **App Service**
   (OS: Linux, runtime: .NET 8).
2. В **Deployment Center** подключите GitHub и выберите репозиторий/ветку `main`,
   либо опубликуйте прямо из Visual Studio (Publish → Azure).
3. В настройках приложения задайте переменную `WEBSITES_PORT` (код уже учитывает
   `WEBSITES_PORT`, см. `Server/Program.cs`).

> Примечание: API-ключи Яндекса сейчас захардкожены в коде (см. ниже).
> Для продакшена их стоит вынести в переменные окружения.

## Известные ограничения (следующие шаги)

- **API-ключи Яндекса** (карты и геокодер) захардкожены в `wwwroot/index.html`
  и `Controllers/PlacemarksController.cs`. Стоит вынести в `appsettings.json` /
  User-Secrets и не коммитить.
- Загруженные фото хранятся локально на диске сервера (не в БД и не в S3);
  на бесплатном хостинге сбрасываются при пересборке. Для продакшена стоит
  вынести в облачное хранилище.
