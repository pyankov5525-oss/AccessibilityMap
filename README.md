# ♿ Карта доступности (AccessibilityMap)

Веб-приложение на **Blazor WebAssembly (hosted, .NET 8)** для оценки доступности
объектов городской среды (вход, ширина дверей, пути, санитарная зона, информация,
парковка, персонал) с отображением на **Яндекс.Картах**.

Каждый объект оценивается по 7 критериям (0–3 балла, итог 0–21) и получает
уровень доступности:

| Баллы | Уровень | Цвет |
|-------|---------|------|
| ≥ 18  | Полностью доступно | 🟢 green |
| 9–17  | Частично доступно | 🟡 gold |
| < 9   | Недоступно | 🔴 red |

## Стек

- **Blazor WebAssembly** (клиент) + **ASP.NET Core** (сервер, хостинг WASM)
- **SQLite** через **Entity Framework Core**
- **Яндекс.Карты 2.1** (JS-interop) + геокодер Яндекса
- CORS — разрешён любой origin (для локальной разработки WASM↔API)

## Возможности

- Карта с метками, раскрашенными по уровню доступности.
- Весь интерфейс (фильтры, кнопка добавления, статистика, форма объекта) —
  поверх самой Яндекс.Карты, а не в отдельной колонке сайта.
- Клик по карте в «режиме доступности» → форма добавления объекта
  (адрес подставляется через обратный геокодинг).
- Редактирование и удаление объектов прямо из балуна на карте.
- Поиск адреса через геокодер с перемещением карты.
- Фильтры по уровню доступности (Все / 🟢 / 🟡 / 🔴) и счётчик статистики.
- Страница **«Список»** (`/places`): таблица со всеми объектами, фильтрами,
  поиском и удалением; кнопка «Изменить» открывает форму на карте (`/?edit=<id>`).
- Сид-данные (объекты Воронежа) при первом запуске.
- Загрузка фотографий объектов (файлы хранятся в `wwwroot/uploads`).

## Запуск

```bash
dotnet build
dotnet run --project Server
```

Откройте адрес, который напечатает консоль (обычно `http://localhost:5000`).
Swagger API доступен в режиме разработки по `/swagger`.

База (`accessibility.db`, SQLite) создаётся и наполняется демо-данными автоматически.

## Структура

```

```

## Публикация (в публичный доступ)

Приложение — полноценный ASP.NET Core сервер (API + отдача Blazor WASM), поэтому
для публикации нужен хостинг, умеющий запускать .NET, а не только статику.

### Вариант 1. Render.com (бесплатно, через Docker)

1. Создайте аккаунт на https://render.com и подключите GitHub-репозиторий
   `pyankov5525-oss/AccessibilityMap`.
2. New → **Web Service** → выберите репозиторий.
3. Environment: **Docker** (в репозитории уже есть `Dockerfile`).
4. Остальное — по умолчанию (план Free).
5. Deploy. Через 1–2 минуты появится URL вида
   `https://accessibilitymap.onrender.com` — его и дайте руководителю.

Сервер сам берёт порт из переменной окружения `PORT` (см. `Server/Program.cs`).
База SQLite и загруженные фото создаются в файловой системе хоста; на бесплатном
тарифе они сбрасываются при пересборке/перезапуске (демо-данные Воронежа подсеются заново).

### Вариант 2. Azure App Service (бесплатный тариф F1)

1. Создайте бесплатную подписку Azure и ресурс **App Service**
   (OS: Linux, runtime: .NET 8).
2. В **Deployment Center** подключите GitHub и выберите репозиторий/ветку `main`,
   либо опубликуйте прямо из Visual Studio (Publish → Azure).
3. В настройках приложения задайте переменную `WEBSITES_PORT` (код уже учитывает
   `WEBSITES_PORT`, см. `Server/Program.cs`).

> Примечание: API-ключи Яндекса сейчас захардкожены в коде (см. ниже).
> Для продакшена их стоит вынести в переменные окружения.

## Известные ограничения (следующие шаги)

- **API-ключи Яндекса** (карты и геокодер) захардкожены в `wwwroot/index.html`
  и `Controllers/PlacemarksController.cs`. Стоит вынести в `appsettings.json` /
  User-Secrets и не коммитить.
- Нет аутентификации/авторизации (любой может добавлять/удалять объекты).
- База данных — локальный SQLite (для продакшена лучше PostgreSQL).
